ВИМОГИ ДО ПРОФІЛІВ БЕЗПЕКИ ДЛЯ ІНФОРМАЦІЙНИХ СИСТЕМ СУДОВОЇ ВЛАДИ УКРАЇНИ
(на основі НД ТЗІ 3.6-006-24 та Наказів Адміністрації Держспецзв'язку №419 від 02.07.2025)

1. ЗАГАЛЬНІ ПОЛОЖЕННЯ ТА ОБҐРУНТУВАННЯ
Цей документ визначає ієрархічну структуру профілів безпеки для інформаційних, електронних комунікаційних та інформаційно-комунікаційних систем органів судової влади (Державна судова адміністрація України). Профілі базуються на національних базових профілях обробки конфіденційної (відкритої) та службової інформації.

2. ДЕРЕВО ОБ'ЄКТНИХ ІДЕНТИФІКАТОРІВ (OID) СУДОВОЇ ВЛАДИ
Для фіксації профілів безпеки в сертифікатах (розширення certificatePolicies) виділено простір OID: 1.2.804.3.1.2

1.2.804.3.1.2.1 - Базовий профіль безпеки (Level 1)
1.2.804.3.1.2.2 - Галузевий профіль безпеки (Level 2)
1.2.804.3.1.2.3 - Цільовий профіль безпеки вищих судів (Level 3)
1.2.804.3.1.2.3.1 - Велика Палата Верховного Суду
1.2.804.3.1.2.3.2 - Касаційний адміністративний суд
1.2.804.3.1.2.3.3 - Касаційний господарський суд
1.2.804.3.1.2.3.4 - Касаційний кримінальний суд
1.2.804.3.1.2.3.5 - Касаційний цивільний суд
1.2.804.3.1.2.4 - Цільовий профіль безпеки вищих спеціалізованих судів (Level 3)
1.2.804.3.1.2.4.1 - Вищий суд з питань інтелектуальної власності
1.2.804.3.1.2.4.2 - Вищий антикорупційний суд
1.2.804.3.1.2.4.3 - Спеціалізований окружний адміністративний суд
1.2.804.3.1.2.4.4 - Спеціалізований апеляційний адміністративний суд
1.2.804.3.1.2.5 - Цільовий профіль судів (Level 3)
1.2.804.3.1.2.5.1 - Цільовий профіль безпеки місцевих судів (місцеві загальні, господарські, окружні, адміністративні) судів (Level 4)
1.2.804.3.1.2.5.2 - Цільовий профіль безпеки апеляційних судів (Level 4)
1.2.804.3.1.2.6 - Цільовий профіль безпеки органів та установи в системі правосуддя (Level 3)
1.2.804.3.1.2.6.1 - ДСА (Level 4)
1.2.804.3.1.2.6.2 - ТУ ДСА, допоміжні установи (Level 4)
1.2.804.3.1.2.6.3 - Рада суддів України - орган суддівського самоврядуванння (Level 4)
1.2.804.3.1.2.6.4 - ВРП - орган суддівського врядування (Level 4)
1.2.804.3.1.2.6.5 - ВККСУ - орган суддівського врядування (Level 4)
1.2.804.3.1.2.6.6 - Національна школа суддів України (Level 4)
1.2.804.3.1.2.6.7 - ГРД та ГРМЕ - громадські ради (Level 4)
1.2.804.3.1.2.6.8 - Служба судової охорони (Level 4)

Додатково в межах базового профілю визначено гілку параметрів безпеки (SPE - Security Profile Extensions) для відображення конкретних контролів згідно NIST SP 800-53:
1.2.804.3.1.2.1.100.<Family_ID>.<Control_ID> - конкретні вимоги безпеки (напр., MP-6 має OID 1.2.804.3.1.2.1.100.10.6).

3. КОДИ РЕКВІЗИТІВ ТА АТРИБУТИВНА ІНФОРМАЦІЯ ЗГІДНО НД ТЗІ 3.6-006-24

3.1. Рівень 1: Базовий профіль безпеки (OID: 1.2.804.3.1.2.1)
Охоплює мінімальні вимоги для систем, де обробляється відкрита або конфіденційна інформація.
- Управління доступом (AC-2, AC-3, AC-4, AC-5, AC-6, AC-7, AC-8, AC-11, AC-12, AC-17, AC-18, AC-19, AC-20, AC-22): 
  - AC-2: Управління обліковими записами (блокування через 24 години).
  - AC-7: Обмеження кількості невдалих спроб входу.
  - AC-11: Блокування пристрою після 30 хвилин бездіяльності.
- Обізнаність та навчання (AT-2, AT-3): Навчання з підвищення обізнаності мінімум раз на рік.
- Аудит та підзвітність (AU-2, AU-3, AU-5, AU-6, AU-7, AU-8, AU-9, AU-11, AU-12):
  - AU-6: Огляд та аналіз журналу аудиту щонайменше щотижня.
  - AU-8: Використання позначок часу (UTC) для записів аудиту.
- Управління конфігурацією (CM-2, CM-3, CM-4, CM-5, CM-6, CM-7):
  - CM-6: Налаштування базової конфігурації за принципом мінімальних привілеїв.
  
(Детальну ієрархію окремих елементів - Security Profile Extensions - наведено в програмній архітектурі CA, модуль CA.SPE).

3.2. Рівень 2: Галузевий профіль безпеки (OID: 1.2.804.3.1.2.2)
Додаткові вимоги до базового профілю для систем судової влади (ЄСІТС, АСДС).
- Ідентифікація та автентифікація: Обов'язкове використання кваліфікованого електронного підпису (КЕП) для авторизації в системі та підписання процесуальних документів.
- Захист інформації: Шифрування каналів зв'язку між підсистемами ЄСІТС.
- Резервне копіювання: Щоденне резервне копіювання баз даних судових рішень.

3.3. Рівень 3: Цільові профілі безпеки вищих судів (OID: 1.2.804.3.1.2.3)
Включає Велику Палату ВС та касаційні суди (адміністративний, господарський, кримінальний, цивільний - 1.2.804.3.1.2.3.1 ... 1.2.804.3.1.2.3.5).
- Багатофакторна автентифікація для всіх облікових записів.
- Кластеризація та висока доступність серверного обладнання.
- Підвищені вимоги до конфіденційності та цілісності остаточних рішень касаційної інстанції.

3.4. Рівень 3: Цільові профілі вищих спеціалізованих судів (OID: 1.2.804.3.1.2.4)
Включає ВСПІВ, ВАКС та спеціалізовані адміністративні суди (1.2.804.3.1.2.4.1 ... 1.2.804.3.1.2.4.4).
- Розширений аудит доступу до справ із грифом "Цілком таємно" (для ВАКС).
- Використання апаратних модулів безпеки (HSM) для генерації та зберігання ключів.
- Ізоляція мережевих сегментів.
- Захист комерційної таємниці учасників процесу шляхом шифрування доказів (для ВСПІВ).

3.5. Рівень 3: Цільовий профіль судів та Рівень 4: Апеляційні та місцеві суди (OID: 1.2.804.3.1.2.5)
- 1.2.804.3.1.2.5.1 Місцеві суди: Стандартизований захист робочих місць суддів і секретарів. 
- 1.2.804.3.1.2.5.2 Апеляційні суди: Захист робочих місць, а також захист каналів зв'язку для відеоконференцзв'язку.

3.6. Рівень 3: Цільовий профіль безпеки органів та установ у системі правосуддя (OID: 1.2.804.3.1.2.6)
Включає ДСА, ТУ ДСА, ВРП, ВККСУ, Раду суддів, НШСУ, ГРД, ГРМЕ, Службу судової охорони.
- Резервування каналів зв'язку та захист баз даних фінансово-господарського характеру.
- Розмежування доступу до реєстрів на етапах кадрового відбору (для ВККСУ, ВРП).
- Захист спеціальних каналів комунікацій (для Служби судової охорони).

4. ІМПЛЕМЕНТАЦІЯ У СЕРТИФІКАТАХ CA
Вказані OID імплементуються у системі генерації сертифікатів CA як значення розширень certificatePolicies (OID 2.5.29.32) та спеціалізованих розширень для Security Profile Extensions, що дозволить однозначно ідентифікувати рівень довіри та профіль безпеки, якому відповідає кінцевий вузол або користувач. Архітектурно це відображено у модулях CA.CP (Certificate Policies) та CA.SPE (Security Profile Extensions).
